以下是详细的大纲，可直接交付开发者按模块实现完整的 Android Kotlin 版 GitHub 一键上传工具。

---

## 项目总体设计目标

- **名称**：GitHub Uploader  
- **功能**：使用 Token 认证，管理仓库，上传/打包文件夹/ZIP，预置工作流模板，下载仓库 ZIP，剪贴板片段管理，配置柔性（JSON→SharedPreferences），自适应字体等。
- **架构**：MVVM + Repository，Kotlin 协程，Jetpack Compose（推荐）或 ViewBinding+Fragment 亦可按此大纲调整。
- **最低 SDK**：24 (Android 7.0)，目标 SDK 34

---

## 1. 技术栈与依赖

| 组件 | 方案 |
|------|------|
| 网络 | Retrofit2 + OkHttp + Moshi/Gson |
| 异步 | Kotlin Coroutines + LiveData/StateFlow |
| 本地存储 | SharedPreferences (DataStore) 存储 Token/配置，可单独 JSON 文件保存片段 |
| UI 层 | Jetpack Compose (Material3) |
| 导航 | Jetpack Compose Navigation |
| DI | 手动依赖注入或 Koin（若简单，可用单例） |
| 文件选择 | ActivityResultContracts (OpenDocument, CreateDocument, OpenDocumentTree) |
| 剪贴板 | ClipboardManager |
| 下载 ZIP | DownloadManager 或自定义 Retrofit 下载+SAF |
| 打包 ZIP | java.util.zip |

**关键依赖版本示例**：
```
retrofit = "2.9.0"
okhttp = "4.12.0"
moshi = "1.15.0"
compose-bom = "2024.02.00"
```

```kotlin
data class GitHubUser(val login: String, val name: String?, val email: String?, val avatarUrl: String)
```

## 4. 网络层接口 (GitHubApiService)

```kotlin
interface GitHubApiService {
    @GET("user")
    suspend fun getAuthenticatedUser(): GitHubUser

    @GET("user/repos")
    suspend fun listUserRepos(@Query("per_page") perPage: Int = 100, @Query("sort") sort: String = "updated"): List<GitHubRepo>

    @GET("repos/{owner}/{repo}/contents/{path}")
    suspend fun getRepoContents(@Path("owner") owner: String, @Path("repo") repo: String,
                                @Path("path") path: String, @Query("ref") branch: String): List<RepoContentItem>

    @GET("repos/{owner}/{repo}/contents/{path}")
    suspend fun getFileDetails(@Path("owner") owner: String, @Path("repo") repo: String,
                               @Path("path") path: String, @Query("ref") branch: String): RepoContentItem?

    @PUT("repos/{owner}/{repo}/contents/{path}")
    suspend fun createOrUpdateFile(@Path("owner") owner: String, @Path("repo") repo: String,
                                   @Path("path") path: String, @Body body: FileUpdateRequest): FileUpdateResponse

    @POST("user/repos")
    suspend fun createRepository(@Body body: CreateRepoRequest): GitHubRepo

    @GET("repos/{owner}/{repo}/zipball/{branch}")
    @Streaming
    suspend fun downloadRepoZip(@Path("owner") owner: String, @Path("repo") repo: String,
                                @Path("branch") branch: String): ResponseBody
}
```

Token 通过 OkHttp Interceptor 添加到 Header：  
`Authorization: token {token}`

## 5. 本地持久化 (PreferencesManager)

使用 SharedPreferences 或 DataStore 存储：

- `token: String`
- `branch: String` (默认 "main")
- `repo_url: String`
- `create_new: Boolean`
- `new_repo_name, desc, private`
- `upload_default_unpack/build: Boolean`
- `build_branch, java_version, java_home, gradle_version, build_type`
- `folder_exclude_patterns: Set<String>` (用逗号分隔或单独key)
- `snippets_json: String` (JSON序列化最多10个 Snippet)
- `font_scale: Float, dialog_scale: Float` (保存缩放比例，可选)

提供同步/挂起函数读写。

## 6. 核心业务逻辑 (domain 层)

### 6.1 UploadManager

- `fun createRepoAndSwitchMode(...)`  
  调用 API 创建仓库 -> 更新本地仓库URL，切换模式为已存在，刷新仓库列表。
- `fun uploadFilesSequence(...)`  
  并行/顺序上传：
  - 若勾选，上传 unpack.yml (从模板生成)
  - 若勾选，上传 build.yml (使用 TemplateGenerator)
  - 遍历自定义 YML 列表，读取文件内容 -> Base64 -> 上传至 `.github/workflows/`
  - 若有 ZIP 文件，上传到仓库根目录。
  - 处理文件 SHA（若存在则传递）
  - 捕获异常并提供日志回调。

### 6.2 TemplateGenerator

- `fun generateUnpackYml(): String` 返回常量模板（即上面 UNPACK_YML_TEMPLATE）
- `fun generateBuildYml(buildBranch: String, javaVersion: String, gradleVersion: String, buildType: String): String`  
  将 BUILD_YML_TEMPLATE 中的占位符替换，如 `{{BRANCH}}` → buildBranch，`{{BUILD_TYPE_CAPITALIZED}}` → buildType.capitalize()。

### 6.3 ZipPacker

- `fun packFolder(contextUri: Uri, excludePatterns: List<String>, protectedFiles: Set<String>): File`  
  - 使用 DocumentFile 或 File API（根据用户选择的 URI，兼容 SAF）。  
  - 递归遍历，应用 fnmatch 规则。保护文件 `.gitignore`, `build.gradle` 等不能排除。  
  - 生成临时 ZIP 文件，返回其路径。  
  - 可考虑使用 `contentResolver` 处理 SAF。

### 6.4 ClipboardManager (与 Android 剪贴板交互)

- `fun getSnippets(): List<Snippet>`
- `fun updateSnippet(index: Int, name: String, content: String)`
- `fun copyToClipboard(context: Context, text: String)` ：使用 `ClipboardManager`。
- 管理最多10个片段，保存在 preferences。

## 7. UI 设计与功能拆解（Compose 示例）

### 7.1 主屏幕 (MainActivity + BottomNav/ViewPager)

底部导航栏（或顶栏Tab）：

1. **操作** (上传主流程)
2. **仓库内容** (文件树 + 下载)
3. **剪贴板**
4. **设置** (使用单独的屏幕或弹出式底部菜单，按大纲建议独立屏幕)

### 7.2 认证部分 (AuthScreen or TopBar)

- 输入 Token（密码可见切换），验证。
- 显示用户头像、用户名。
- 若 Token 未配置，所有其他功能禁用引导。

### 7.3 操作流程 (UploadScreen)

**三段式卡片布局：**

1. **仓库设置卡片**
   - 切换按钮：“使用现有仓库” / “创建新仓库”
   - 现有模式：文本输入框 + 快速搜索下拉（AutoComplete）
     - 从列表过滤，选择后显示仓库地址。
   - 新建模式：仓库名称、描述、私有开关。

2. **文件选择卡片**
   - 按钮：选择 YML 文件 / 选择文件夹 / 清空列表
   - YML 文件列表（LazyColumn，支持删除、长按菜单）
   - ZIP 选择展示（显示文件名，支持移除）；文件夹打包后显示为 ZIP。

3. **上传按钮**
   - 执行前弹出确认对话框（列出即将上传内容）。
   - 显示日志区域，实时追加（使用 `Flow` 在视图中更新）。

### 7.4 日志面板

- 可滚动的 `LazyColumn`，每项显示时间戳+消息。
- 上传过程中和加载仓库内容时日志同步输出。
- “清除”按钮。

### 7.5 仓库内容 (RepoContentScreen)

- 输入仓库 URL 或自动同步当前选择的仓库。
- 加载顶层目录，树形显示（可使用可折叠列表）。
- 支持“刷新”和“下载 ZIP”
  - 下载 ZIP 时使用 DownloadManager 或自行下载后通过 SAF 保存。
  - 下载进度可选。

### 7.6 剪贴板片段 (SnippetScreen)

- 最多10个片段，每个带名称和内容。
- 下拉选择后，在文本框中预览。
- 按钮：“复制到剪贴板”、“管理片段”打开底部编辑表单。
- 管理界面：列表+编辑区，可保存修改。

### 7.7 设置 (SettingsScreen)

**标签页式设计：**

1. **界面显示**
   - 字体缩放滑块 (1.0 ~ 2.0)，影响应用内字体大小（`MaterialTheme.typography`动态缩放）。
   - 对话框/间距缩放滑块 (0.8 ~ 2.0)，控制 `dp` 基础乘数。
   - 预设按钮：标准、大字、超大。
   - 提示重启生效。

2. **基本设置**
   - Token 修改（覆盖主界面）
   - 默认分支
   - 默认 Workflow 开关（unpack、build）
   - 新仓库描述、私有

3. **Build 配置**
   - 触发分支选择（main/master/自定义）
   - Java版本、JAVA_HOME 自定义、Gradle 版本
   - 构建类型（debug/release）

4. **打包排除**
   - 多行文本编辑，每行一个 pattern，默认提供常见排除项。
   - 保存至本地。

5. **关于/剪贴板提示** 可合并。

## 8. 详细交互与线程处理

- 所有网络调用在 `viewModelScope` 内启动协程。
- 文件操作（读 YML、打包目录）在 `Dispatchers.IO` 上执行。
- UI 状态通过 `StateFlow` 驱动 Compose 重组。
- 使用 `Snackbar` 提示错误/成功。
- 上传过程中可展示进度条（文件数量），但 API 批量上传不易获取进度的可仅显示步骤日志。

## 9. 文件选择与打包实现细节

- **选择 YML 文件**：`ActivityResultContracts.OpenDocument` 筛选 `application/x-yaml` 或 `*/*`。
- **选择文件夹并打包**：
  - Android 11+ 推荐使用 `Intent(Intent.ACTION_OPEN_DOCUMENT_TREE)` 获取 URI。
  - 通过 `DocumentFile.fromTreeUri` 递归遍历。
  - 忽略列表模式匹配：`fnmatch` 可用简单的 `Pattern.compile(wildcardToRegex(pattern))`。
  - 保护文件 `.gitignore`, `build.gradle`, `settings.gradle`, `gradle.properties`, `gradlew`, `gradlew.bat` 硬编码不排除。
  - 生成 ZIP 暂存于应用缓存目录。
- **下载仓库 ZIP**：
  - 使用 `DownloadManager.Request` 或自行 Retrofit 流式下载后通过 `ActivityResultContracts.CreateDocument("application/zip")` 保存。

---

## 10. 剪贴板片段持久化

- 使用 Gson/Moshi 将 `List<Snippet>` 序列化为 JSON 字符串存入 SharedPreferences。
- 最多限制 10 个，提供添加/删除逻辑。
- 复制：`ClipboardManager.setPrimaryClip(ClipData.newPlainText("label", content))`。

---

## 11. 生命周期与配置变更

- 使用 ViewModel 保存状态，避免旋转丢失。
- 打包文件夹任务可能耗时，考虑用 `Job` 可取消。
- 窗口尺寸变化由 Compose 自动适配，无需特殊处理。
